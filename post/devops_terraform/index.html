CLI

操作工作目录

// 初始化目录，会根据配置下载和安装provider.
terraform init

terraform get

操作tf文件

// 格式化文件
terraform fmt -recursive

// 验证文件
terraform validate 

// 可以通过终端验证函数.
terraform console

操作基础设施:

terraform plan
terraform plan -out test.tfplan

terraform apply -auto-approve
terraform apply test.tfplan

terraform destroy

检查基础设施:

terraform show
terraform graph
terraform output
terraform state list/show

认证:

terraform login
terraform logout

data source: 获取服务的参数

module

管理provider上的模块/服务.定义了input, resource/data source, output的就可以认为是module.

resources

resources: 创建服务的参数

meta-arguments

  • depends_on
  • count
  • for_each
  • provider
  • lifecycle

lifecycle

用来控制资源的创建、更新和删除行为。

connection

用来定义如何连接到远程资源，通常用于provisioner.

provisioner

特殊的resource，可以在本地或远程机器做一些准备工作。也就是在资源创建后执行一些操作.

支持：

  • file
  • local-exec
  • remote-exec

Variable

https://www.terraform.io/language/values/variables

input variable: 输入变量

local variable: 本地变量

output variable: 输出参数

子模块无法引用父模块的local变量， resource属性，input变量。但是可以通过output参数引用。

父模块无法引用子模块的local变量，resource属性。但是可以通过output参数引用，而且父模块可以通过input变量传值给子模块。

函数

workspace

Workspace 是 Terraform 用来隔离状态文件（state）的一种机制，允许你在同一个配置文件中管理多个环境（如 dev、staging、prod）而不会相互干扰。

terraform workspace new dev
terraform workspace list
terraform select <name>

使用terraform.workspace作为参数.
